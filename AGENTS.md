# AGENTS.md

Инструкции для AI-агентов в репозитории aiproxy — мульти-провайдерный AI API прокси, форк copilot2api v0.4.0.

## Идентификация проекта

- **База:** [copilot2api v0.4.0](https://github.com/whtsky/copilot2api). Это форк. За документацией к базовому проекту — в оригинальный репозиторий.
- **Модуль:** `github.com/whtsky/copilot2api`
- **Рантайм:** Docker Compose — сервис `aiproxy`, контейнер `aiproxy`, образ `aiproxy:local`, порт `127.0.0.1:8081` → `7777`.
- **Роутинг моделей:** по префиксу. `copilot-gpt-5.4` → Copilot, `deepseek-v4-pro` → DeepSeek, и т.д.

## Документация

- `README.md` — пользовательская документация: быстрый старт, конфигурация, примеры клиентов
- `CHANGELOG.md` — изменения нашего форка (начиная с v1.0.0)
- `ARCHITECTURE.md` — структура пакетов, поток запросов, логика роутинга, жизненный цикл провайдеров
- `CLAUDE.md` — симлинк на этот файл

## Правила

- При изменении поведения, API, роутинга или user-visible настроек обновляй `CHANGELOG.md` в том же изменении, в секции `## [Unreleased]`.
- Записи в changelog: короткие сгруппированные пункты под заголовками `Новые возможности`, `Исправления`, `Критические изменения` и т.д.
- Не включай внутренние рефакторинги, тесты или CI-изменения, если они не затрагивают пользователей.
- Не удаляй старые заметки о релизах без явного запроса.
- Все настройки провайдеров — через `providers.json`. Никаких env-переменных для отдельных провайдеров.
- `proxy_host` провайдера использует `upstream.ParseProxyURL` — поддерживает `http://`, `socks5://`, `socks5h://`. Глобальной переменной `PROXY` нет.
- Аутентификация провайдера: `oauth` (device flow, токены в `~/.config/copilot2api/`) или `api_key` (статический ключ в json). Взаимоисключающие — проверяется при старте.
- `providers.json` обязателен. Никакого устаревшего env-var fallback.

## Стиль коммитов

- [Conventional Commits](https://www.conventionalcommits.org/): `тип: описание строчными`
- Типы: `feat`, `fix`, `docs`, `refactor`, `ci`, `test`, `chore`
- Тема: строчными, повелительное наклонение, до 72 символов, без точки в конце

## Верификация

- `go build ./... && go vet ./... && go test ./...` — полный цикл сборки и тестов
- `docker compose build` — Docker-образ `aiproxy:local`
- При отладке: проверяй реальные пути кода, конфиг в репозитории, логи и payload запросов прежде чем предлагать причины
